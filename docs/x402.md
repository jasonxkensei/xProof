# x402 on Base — Agent Payment Guide

xProof supports the [x402 HTTP Payment Protocol](https://x402.org), allowing any AI agent to certify files on-chain without an account, API key, or prior registration. Payment is in **USDC on Base** (eip155:8453).

## Why x402?

| | x402 | API Key |
|---|---|---|
| Account required | No | Yes |
| Payment | Per request (USDC on Base) | Prepaid credits or wallet |
| Setup time | Zero | Minutes |
| Best for | Autonomous agents, one-shot tasks | High-volume pipelines |

The two models coexist. An agent can use x402 for ad-hoc certifications and switch to an API key once volume justifies it.

---

## The Flow

x402 follows the standard HTTP 402 challenge-response pattern.

### Step 1 — Send a request without credentials

```bash
curl -i -X POST https://xproof.app/api/proof \
  -H "Content-Type: application/json" \
  -d '{
    "file_hash": "e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855",
    "filename": "agent_output.pdf"
  }'
```

### Step 2 — Parse the 402 challenge

The server responds with `HTTP 402 Payment Required` and a JSON body:

```json
HTTP/1.1 402 Payment Required

{
  "x402Version": 1,
  "accepts": [
    {
      "scheme": "exact",
      "price": "$0.05",
      "network": "eip155:8453",
      "asset": "USDC",
      "payTo": "0x...",
      "maxTimeoutSeconds": 60,
      "description": "xproof single file certification"
    }
  ],
  "resource": "https://xproof.app/api/proof",
  "description": "xproof single file certification",
  "mimeType": "application/json"
}
```

Your agent reads `accepts[0]` to extract `network`, `price`, `payTo`, and `asset`.

### Step 3 — Pay on Base and retry

Using an x402-compatible facilitator (e.g., [x402.org/facilitator](https://www.x402.org/facilitator)), your agent:

1. Transfers 0.05 USDC to `payTo` on Base Mainnet (eip155:8453)
2. Gets a signed payment receipt (base64-encoded JSON)
3. Retries the original request with the `X-Payment` header

```bash
curl -X POST https://xproof.app/api/proof \
  -H "Content-Type: application/json" \
  -H "X-Payment: eyJ4NDAyVmVyc2lvbiI6MSwidHlwZSI6ImV4YWN0IiwicGF5bG9hZCI6ey4uLn19" \
  -d '{
    "file_hash": "e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855",
    "filename": "agent_output.pdf"
  }'
```

### Step 4 — Receive the proof

```json
HTTP/1.1 200 OK

{
  "proof_id": "prf_abc123",
  "file_hash": "e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855",
  "filename": "agent_output.pdf",
  "tx_hash": "0xabc...",
  "blockchain_status": "confirmed",
  "verify_url": "https://xproof.app/proof/prf_abc123",
  "certificate_url": "https://xproof.app/certificate/prf_abc123"
}
```

## Python Example

Using the xProof Python SDK with x402:

```python
import hashlib
import base64
import json
import httpx

XPROOF_API = "https://xproof.app"

def sha256_file(path: str) -> str:
    h = hashlib.sha256()
    with open(path, "rb") as f:
        for chunk in iter(lambda: f.read(8192), b""):
            h.update(chunk)
    return h.hexdigest()

def certify_with_x402(file_path: str, facilitator_client) -> dict:
    file_hash = sha256_file(file_path)
    filename = file_path.split("/")[-1]

    payload = {"file_hash": file_hash, "filename": filename}

    # Step 1: Initial request — will get 402
    r = httpx.post(f"{XPROOF_API}/api/proof", json=payload)

    if r.status_code == 402:
        requirements = r.json()

        # Step 2: Pay via facilitator (agent's x402 client)
        payment_payload = facilitator_client.pay(requirements["accepts"][0])
        x_payment = base64.b64encode(json.dumps(payment_payload).encode()).decode()

        # Step 3: Retry with payment
        r = httpx.post(
            f"{XPROOF_API}/api/proof",
            json=payload,
            headers={"X-Payment": x_payment}
        )

    r.raise_for_status()
    return r.json()
```

## Agent Compatibility

Any agent with x402 support works out of the box:

- **Conway / OpenClaw agents** — native x402 support
- **Custom agents** — implement the 402 challenge-response loop above
- **MCP-compatible agents** — use the [xProof MCP tool](./mcp.md) which handles x402 internally

No MultiversX wallet is required on the agent side. xProof handles blockchain anchoring on MultiversX independently of the Base payment.

---

## Architecture Note

xProof uses two chains with distinct roles:

- **Base** — handles payments via x402 in USDC. Where agents operate today.
- **MultiversX** — handles on-chain anchoring. 6-second finality, $0.001/tx cost.

The two chains are not in competition. Agents pay on Base, proofs are recorded on MultiversX.
